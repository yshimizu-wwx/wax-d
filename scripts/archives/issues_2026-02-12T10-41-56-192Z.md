[タイトル]: [Bug] mastersテーブルへのデータ登録時の権限エラー (Permission Denied) の解消

Type: Bug
Priority: High
Milestone: v0.1

概要
業者画面の品目一覧（マスター管理）にて、新しい品目（例：「りんご」）を登録しようとすると「permission denied for table masters」というエラーが発生し、保存に失敗する。

現状の課題・背景
DBレベルでの書き込み権限が不足している。文字起こし内容から、データベース（恐らくSupabase等のPostgreSQL）の masters テーブルに対して、現在のログインユーザー（業者ロール）に INSERT 権限が付与されていない、もしくはRLS (Row Level Security) ポリシーによって拒否されている可能性が高い。

修正・実装内容
[ ] masters テーブルのRLSポリシーを確認し、業者ユーザーがデータを挿入（INSERT）できるポリシーを追加・修正する。
[ ] データベースのユーザーロール（Role）に masters テーブルへのアクセス権限が適切に付与されているか確認する。
[ ] フロントエンドのエラーハンドリングを強化し、権限不足の際にユーザーに分かりやすいメッセージを表示するように修正する。

関連ファイル (推測)

supabase/migrations/xxxx_create_masters_table.sql (RLS定義)

lib/supabase.ts (DBクライアント設定)

src/features/master/hooks/useCreateItem.ts (登録ロジック)

受入基準 (Doneの定義)
[ ] 業者画面から新しい品目を入力し、エラーなく masters テーブルにデータが保存されること。
[ ] 保存成功後、一覧画面に正しく反映されること。

[タイトル]: [UI/UX] 品目入力フィールドのテキスト視認性改善（白背景・白文字問題の修正）

Type: UI-UX
Priority: Medium
Milestone: v0.1

概要
品目一覧の検索または登録時の入力フォームにおいて、背景色と文字色が共に白（もしくは極めて近い色）になっており、入力内容が判別できない。

現状の課題・背景
フォームのスタイリング定義において、ダークモード/ライトモードの切り替え設定の不整合、またはコンポーネント固有のカラー指定ミスにより、入力中のテキストカラーが適切に設定されていない。ユーザーが「何を入力しているか分からない」という致命的な操作性の低下を招いている。

修正・実装内容
[ ] 当該入力フィールドの color プロパティを修正し、背景色とのコントラストを確保する。
[ ] フォーカス時、ホバー時のスタイリングを確認し、一貫性のあるUIに修正する。
[ ] グローバルなテーマ設定（Tailwind CSSのコンフィグ等）が正しく適用されているか確認する。

関連ファイル (推測)

src/components/ui/Input.tsx (共通コンポーネント)

src/features/master/components/ItemForm.tsx (品目登録フォーム)

src/styles/globals.css (または tailwind.config.js)

受入基準 (Doneの定義)
[ ] 品目一覧の入力フィールドにテキストを入力した際、はっきりと文字が読み取れる（コントラスト比の確保）。
[ ] プレースホルダーと入力後のテキストが正しく区別できる色になっていること。
